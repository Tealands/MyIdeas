
# EPARK Extension: 「Time-Sync Queue（タイムシンク・キュー）」導入提案書

作成日:2026/3/26

## 1. コンセプト

**「待たせない」の先へ。待ち時間を「消費時間」に変えるデッドライン逆算型システム。**
既存のEPARKが持つ「順番待ち・予約」の基盤に、空港の「搭乗時刻」や遊園地の「イベント時刻」などの**絶対的終止符（デッドライン）**を同期。ユーザーの「遅れられない不安」を解消し、空き時間での回遊・購買を最大化させます。

## 2. ターゲットと活用シーン

EPARKの汎用性を活かし、以下の「時間制約が厳しい」大規模施設をターゲットにします。

| **導入ターゲット**   | **デッドラインの定義** | **待ち時間で創出する価値**               |
| -------------------------- | ---------------------------- | ---------------------------------------------- |
| **国際・国内空港**   | 航空機の搭乗開始時刻         | 免税店での買い物、ラウンジ利用、シャワー・仮眠 |
| **テーマパーク**     | パレード開始、帰宅バス時刻   | 離れたエリアの飲食、お土産購入、ARゲーム体験   |
| **大型展示会・催事** | セミナー開始、閉場時刻       | 離れたブースの回遊、商談予約、限定グッズ購入   |
| **大規模フェス**     | お目当てのアーティスト開演   | グッズ列の仮想化、飲食ブースの事前決済・受取   |

## 3. 提案する追加機能（コア・アップグレード）

* **デッドライン逆算アラート**
  * 航空券や入園チケットのQRコードを読み取り、EPARKの待ち時間データと照合。
  * 「今から並ぶと搭乗に間に合いません」「あと15分あれば、隣のカフェでコーヒーが飲めます」とプッシュ通知。
* **「隙間時間」専用レコメンド**
  * 算出された「自由な待ち時間」内で完結する、周辺店舗のクイックメニューやクーポンを自動配信。
* **インテリジェント・スキップ（優先割当）**
  * デッドラインが迫っているユーザーを、AIが自動で列の前方に繰り上げる、または別ルートへ誘導する動的最適化。

## 4. アイフラッグ（EPARK）への導入メリット

* **新規市場の開拓** : 飲食店・病院中心のモデルから、インフラ・レジャー巨大市場への進出。
* **アクティブユーザーの質的向上** : 「ただ待つ」ユーザーを「施設内で消費する」アクティブな消費者に変容させ、加盟店（空港ビル・パーク運営）への貢献度を高める。
* **データ利活用の深化** : ユーザーが「どの程度の空き時間があれば、どの程度の追加消費を行うか」という行動ログの蓄積。

## 5. 独自の応用場面（その他の提案）

空港や遊園地以外で、この「デッドライン逆算型」が威力を発揮する場面です。

* **免許更新センター・役所** :
* 講習開始時刻や閉庁時間から逆算。待ち時間に周辺の証明写真機やカフェへ誘導。
* **就職博・合同説明会** :
* お目当ての企業説明会が始まるまでの隙間時間を使い、空いている小規模ブースへの訪問を促す（マッチング効率の向上）。
* **サービスエリア（SA・PA）** :
* 「次の目的地への出発希望時間」を設定。混雑するトイレやレストランの順番待ちをしつつ、特産品コーナーでの買い物を推奨。
