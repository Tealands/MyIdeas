# 次世代キャンパス・モビリティ実証実験 企画書

**〜自動運転バスによる学内交通インフラの構築とMaaSの実装に向けて〜**

## 1. 企画の背景と目的

広大なキャンパスにおける移動効率の向上は、学修・研究環境の質に直結する課題です。本プロジェクトでは、最先端の自動運転技術を導入することで、以下の目的を達成します。

* **移動のバリアフリー化：** 勾配や距離による移動負担を軽減し、教職員・学生の学内回遊性を高める。
* **安全性の向上：** 鉄道会社の安全管理ノウハウを導入し、交通事故リスクを低減した交通システムを構築する。
* **リビングラボの実現：** 大学を実証フィールドとして開放し、自動運転技術や人流データの社会実装を加速させる。

---

## 2. 実施体制（産学連携モデル）

各組織の強みを活かした協力体制を構築します。

| **パートナー**            | **主な役割**                                                               |
| ------------------------------- | -------------------------------------------------------------------------------- |
| **本学（主催）**          | フィールド提供、学生・教職員への利用促進、倫理審査、データの学術利用。           |
| **鉄道会社**              | 運行管理システム（ダイヤ編成・遠隔監視）の提供、安全管理基準の策定、乗務員教育。 |
| **研究機関**              | センサー技術（LiDAR/GNSS）の最適化、AIによる障害物回避アルゴリズムの検証。       |
| **車両/システムベンダー** | 自動運転車両（EVバス）の提供、メンテナンス、通信インフラ整備。                   |

---

## 3. 実証実験のフェーズ計画

段階的に難易度を上げ、安全性を確認しながら本格運用へ移行します。

### Phase 1：技術検証（クローズド環境）

* **内容：** 夜間や休日等、歩行者の少ない時間帯での自律走行テスト。
* **着眼点：** 走行ルートの3Dマップ作成、信号連携、通信環境の安定性。

### Phase 2：社会受容性・安全性検証

* **内容：** 限定的なルートでの試験運行。特定のモニター（学生・教職員）による試乗。
* **着眼点：** 混雑時の挙動、歩行者の心理的影響、乗車予約アプリの使い勝手。

### Phase 3：本格実装・学内インフラ化

* **内容：** 授業時間割に合わせた定時運行、およびオンデマンド運行。
* **着眼点：** 維持管理コストの最適化、学内MaaS（他交通機関との連携）。

---

## 4. 走行ルートおよびシステム構成図

キャンパス内の主要拠点（正門、図書館、講義棟、食堂、研究棟）を効率的に結ぶルートを設定します。

* **車両：** 低速電動自動運転バス（レベル4相当を想定）
* **インフラ：** 路車間通信（V2X）用センサー、遠隔監視センター
* **インターフェース：** スマホアプリ（リアルタイム位置確認・予約）

---

## 5. 安全管理およびリスク対策

鉄道会社との協力により、公共交通機関レベルの安全基準を策定します。

1. **遠隔監視体制：** 常に遠隔から車両の状態を確認し、緊急時には即座に手動介入が可能。
2. **物理的対策：** 車両へのセンサー多重化、緊急停止ボタンの設置、車内外へのアラート送出。
3. **法的遵守：** 道路交通法および道路運送法（学内私道・公道それぞれ）に基づいた運行。

---

## 6. 期待される成果

* **教育・研究面：** 実際の走行データを活用したAI・ロボティクス研究の促進。
* **ブランド向上：** 「最先端ITキャンパス」としての対外的アピール、オープンキャンパスでの活用。
* **地域貢献：** 将来的な周辺地域（ラストワンマイル）への自動運転サービス展開のモデルケース。
