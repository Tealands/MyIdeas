# 事業計画案：AI共通一次面接プラットフォーム『Rec-Port（レックポート）』

## 1. 事業コンセプト

**「一度の録画で、全ての一次選考をパスする」**
学生が汎用的な質問に対する回答を一度録画・AI解析し、そのデータを複数の提携企業に「共通一次選考スコア」として提出・使い回しができるプラットフォーム。

## 2. 解決する課題

* **企業側** :
* 重複する「一般的な質問」に費やす数千時間の面接工数。
* 書類選考（ES）だけでは判別できない「コミュニケーション能力」のミスマッチ。
* **学生側** :
* 企業ごとに同じ内容（ガクチカ等）を何度も話す「面接の非効率（タイパの悪さ）」。
* 志望動機が固まる前の「初期段階」での選考落ち。

## 3. サービス構成（機能概要）

### ① 共通質問パッケージ（志望動機を除く）

あえて個別性の強い「志望動機」を排除し、個人の素養・ポテンシャルを測る質問に特化します。

* **自己紹介 / 自己PR** （第一印象・要約力）
* **学生時代に力を入れたこと（ガクチカ）** （行動特性・論理性）
* **困難を乗り越えた経験** （レジリエンス・ストレス耐性）
* **自身の長所・短所とそれへの向き合い方** （客観性・自己改善力）

### ② AI多角解析エンジン

録画データから以下の要素を数値化（スコアリング）し、企業に提示します。

* **構造化スコア** : 結論から話せているか、論理構成は適切か。
* **印象スコア** : 表情、声のトーン、話すスピードなどの非言語情報。
* **信頼性チェック** : 視線検知による「カンペ読み」の有無、AI生成テキストの読み上げ検知。

### ③ データ共有・検索システム

* 学生は一度の撮影で、提携企業（A社、B社、C社...）へ一括、または個別に応募可能。
* 企業は自社の求める「AIスコア（例：論理性がAランク以上）」で学生をフィルタリングし、動画を確認。

## 4. ターゲット・展開戦略

* **初期ターゲット** :
* 応募数が多く、一次選考の自動化ニーズが高い **メガベンチャー・大手IT企業** 。
* 特に「話し方」や「地頭」が重視される **営業職・エンジニア職** 。
* **展開パートナー候補** :
* **日本SHL（玉手箱）** : 適性検査のオプションとして動画スコアを統合。
* **スカウト型サイト（OfferBox等）** : プロフィールに「認定動画」として掲載。

## 5. ビジネスモデル

* **BtoB** :
* システム利用料（月額SaaS）
* 動画閲覧・選考通過ごとの従量課金
* **BtoC** :
* 基本無料
* （オプション）AIによる模擬フィードバック・ブラッシュアップ機能

## 6. 差別化ポイント

| 項目               | 既存の録画面接ツール   | 本プロジェクト (Rec-Port)                    |
| ------------------ | ---------------------- | -------------------------------------------- |
| **主権者**   | 企業（企業ごとに撮影） | **学生（一度撮れば使い回し可能）**     |
| **評価主体** | 人間（人事が全て見る） | **AI（水準未満を自動フィルタリング）** |
| **志望動機** | 必須（個別対応が必要） | **不要（汎用スキルに特化）**           |
