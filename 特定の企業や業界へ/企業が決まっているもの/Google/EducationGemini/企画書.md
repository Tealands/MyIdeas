## 1. 概要

- 本企画の要旨（誰に／何を／どのように提供するかを端的に）

## 2. 背景・課題

- 現状の課題（定量・定性）:
  - 例: 顧客満足度が低下している、取引コストが高い、法規対応が遅れている等
- なぜ今対応が必要か（市場環境、技術、法規の変化など）

## 3. 目的・ゴール

- 短期（6ヶ月）: [具体的な成果物・指標]
- 中期（1年）: [KPIで表現]
- 長期（3年）: [事業のあり方]

※ SMART（具体的・測定可能・達成可能・関連性・期限）で記載する

## 4. ターゲット／市場

- ターゲット顧客（ペルソナ）: 年齢、職業、課題・ニーズ
- 市場規模（概算とソース）:
- 競合（直接・間接）と比較表:
  - 競合A: 強み・弱み
  - 競合B: 強み・弱み

## 5. 提案内容（プロダクト／サービス）

- 概要説明
- コア機能／提供価値
- 差別化ポイント
- 利用フロー（簡単なユーザーシナリオ）

## 6. 事業モデル・収益計画

- 収益の柱（例: サブスクリプション、トランザクション手数料、導入費）
- 価格設定（想定価格帯）
- 想定顧客獲得数／LTVの前提

## 7. 実施計画／スケジュール

- 主要マイルストーン表（例）

| 期間              | マイルストーン | 担当       | 備考 |
| ----------------- | -------------: | ---------- | ---- |
| 2026-01 ~ 2026-03 |       要件定義 | 事業／PM   |      |
| 2026-04 ~ 2026-06 |    開発（PoC） | 開発チーム |      |
| 2026-07 ~ 2026-09 |     検証・改善 | QA / 顧客  |      |
| 2026-10           |       ローンチ | 全体       |      |

## 8. 必要リソース

- 人員: 必要スキルと人数（例: フロント2名、バックエンド2名、PM1名）
- 外注/パートナー: 必要なら名前や役割
- 技術・設備: サーバ、ライセンス、ツール等

## 9. 予算・収支見通し

- 概算（初期費用 / 月次費用）

| 項目           | 金額（円） | 備考          |
| -------------- | ---------: | ------------- |
| 開発人件費     |  8,000,000 | 6ヶ月分見積り |
| サーバ費用     |    200,000 | 年間見積り    |
| マーケティング |  1,000,000 | 初期施策      |
| 合計（概算）   |  9,200,000 |               |

- 売上予測（年）と損益分岐点の想定

## 10. KPI（成功指標）

- 主要KPI: 月間MAU、顧客獲得単価（CAC）、継続率（リテンション）、LTV
- モニタリング方法と頻度

## 11. リスクと対策

- 高リスク項目と想定影響、対策
  - 例: 法規変更 → 法務確認・代替案の事前設計
  - 例: 開発遅延 → スコープ調整／外注活用

## 12. 法務・許認可／備考

- 必要な許認可や契約上の注意点
- 個人情報やセキュリティに関する要件

## 付録（データ・参考資料）

- 詳細な市場データ、調査資料、参考リンク
